## 开发规范

### 代码规范
- **缩进**: Tab缩进
- **命名**: 驼峰命名法（camelCase）
- **引号**: 单引号
- **注释**: 关键业务逻辑必须添加注释

### Git规范
- **分支策略**: Git Flow
  - `main`: 生产分支
  - `develop`: 开发分支
  - `feature/*`: 功能分支
  - `hotfix/*`: 紧急修复分支
- **提交信息**: 采用Conventional Commits规范
  - `feat:` 新功能
  - `fix:` Bug修复
  - `docs:` 文档更新
  - `style:` 代码格式调整
  - `refactor:` 重构
  - `test:` 测试相关
  - `chore:` 构建/工具相关

### API规范
- **基础路径**: `/api/v1`
- **请求格式**: JSON
- **响应格式**: 统一Response类
- **认证方式**: JWT Token
- **状态码**: 遵循HTTP标准状态码
